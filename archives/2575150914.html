音频基础概念

发表于 2019-11-11 分类于 音视频技术 阅读次数：
本文字数： 2.3k 阅读时长 ≈ 2 分钟

声音的物理表现

初中物理，声音是由物体震动而产生的。声音是一种波，声波的三要素是频率、振幅和波形，频率表示音阶的高低，振幅表示响度，波形代表音色。波的频率越高，波长越短，能量损失越少，就能传播的越远。
人类的听力有一个频率范围，大概是20Hz~20kHz，人耳对3~4kHz频率的声音比较敏感，对不在这个范围内的频率，敏感度就会减弱。

音频数字化

音频模拟信号转换为数字信号，大致分为三个过程：采样，量化，编码；

采样

采样就是对模拟信号，在时间轴上对连续的模拟信号进行间隔采集过程。根据采样定理(奈奎斯特定理：为了不失真地恢复模拟信号，采样频率应该不小于模拟信号频谱中最高频率的2倍)，所以采样频率一般为44.1kHz，保证20kHz的音频也能被很好的数字化，保证音频质量不会被降低。44.1kHz表示1秒内，采样44100次。采样的频率越高，声音的还原就越真实和自然。采样如下图所示：

量化

量化就是在幅度轴上，对每一个采样点进行数字化处理。比如用16bit来表示一个采样点，取值范围就是[-32768,32767]，有65536种可能，最终模拟信号在幅度上分为65536层。量化的位数一般有8bit、16bit、32bit，用来表示声音强度的级别。量化如下图所示：

编码

采样完成之后，需要把采样的数据进行存储。PCM(Pulse Code Modulation)数据：音频的裸数据存储格式；描述一段PCM数据需要几个概念：量化格式(sampleFormat)、采样率(sampleRate)、声道数(channel)；

  • 量化格式：音频采样的存储格式，量化的位数(采样位数、位深度、分辨率)一般有8bit、16bit、32bit，用来表示声音强度的级别。把量化得到的结果，以二进制进行存储，有不同的存储方式，分为整形和浮点型。
  • 采样率：每秒对声音模拟信号的采集次数，上面有解释。
  • 声道：当人听到声音的时候，能对声源进行定位，那么通过在不同的位置设置声源，就可以造就出更好的听觉感受。双声道是最常见的类型。

对于声音，还有一个概念来描述音频质量的高低，比特率(码率)，是指单位时间内，传输或者处理比特的数量，比特率是一个速度单位。压缩的音频文件常常用倍速来表示，譬如达到CD音质的MP3是128kbps/44.1kHz。
对于CD音质的音频数据，采样率为44.1kHz，双声道，采样位数16bit，1分钟的存储数据量，大概是：44100 16 2 * 60 / 8 / 1024 = 10.09MB。存储这一段二进制数据，即表示将模拟信号转变成为了数字信号，以后就可以对这段数据进行其他操作了。

麦克风是如何采集声音的：麦克风里有一层碳膜，非常薄而且很敏感。声波是一种纵波，会压缩空气也会压缩这一层碳膜，碳膜在受到挤压时也会发生震动，在碳膜下方有一个电极，碳膜在震动的时候会触及电极，接触的时间长短和频率与声波的震动幅度和频率有关，这样就完成了声音信号到电信号的转换。之后再经过放大电路处理，就可以实施采样量化处理了。

音频的编码

从上面的计算中得知，CD音质的一分钟原始音频数据，数据量有10.09MB，数据量过大，所以在网络上传播的话，需要进行数据压缩编码。压缩编码的基本指标就是压缩比，压缩比通常小于1。压缩算法包括有损压缩和无损压缩。有损压缩，数据解压缩之后，会丢失一部分数据，压缩比越小，丢失的信息就越多；无损压缩，数据解压之后，能得到原始音频数据。压缩编码的原理实际上是去掉冗余信息，冗余信息是指人耳感知不到的音频信息。
下面介绍几种常见的音频编码格式：

WAV编码

PCM是音频原始数据格式，WAV就是在PCM的数据上，再封装一层，而形成新的数据格式。WAV编码有多种实现方式，一种是在PCM数据前面加44字节，来描述PCM的采样率、声道数、数据格式等信息。

无损压缩，音质很好，大量软件都支持；
适用场合：多媒体开发的中间文件，保存音乐和音效素材；

MP3编码

MP3具有不错的压缩比，使用LAME编码（MP3编码的一种实现）的中高码率的MP3文件，听感上很接近WAV文件。

有损压缩，在比特率为128Kbit/s以上表现不错，压缩比比较高，大量软甲和硬件都支持，兼容性好；
适用场合：高比特率下，对兼容性有要求的音乐欣赏；

AAC编码

AAC是新一代音频有损压缩技术，通过一些附加编码技术（比如PS，SBR），衍生出LC-AAC、HE-AAC、HE-AAC v2三种主要的编码格式。

  • LC-AAC：相对于传统的AAC来说，主要应用于中高码率的编码（>=80Kbit/s）；
  • HE-AAC：相当于AAC+SBR，主要应用于中低码率场景的编码（<=80Kbit/s）;
  • HE-AAC v2：相当于AAC+SBR+PS，主要应用于低码率场景的编码（<=48Kbit/s）;

有损压缩，在小于128Kbit/s的码率下表现优异，多用于视频中的音频编码；
适用场合：128Kbit/s以下的音频编码，多用于视频中音频轨的编码；

Ogg编码

是一种非常有潜力的编码，在各种码率下都比较优秀，尤其是中低码率场景下。完全免费，但是在软硬件支持上，情况不够好。

有损压缩，可以比MP3更小的码率实现更好的音质，高中低码率均表现良好，兼容性不够好，流媒体特性不支持；
适用场合：语音聊天的音频消息场景；
